Clarifying Questions

1. Data cadence asymmetry: Marwan said Aura forecasts daily from January, while USDA publishes monthly from May. Should the overlay show Aura's daily updates alongside USDA's monthly snapshots, or align both to monthly for cleaner comparison?

2. Third source — local ministries: Marwan mentioned comparing "USDA forecasts with Aura forecasts and local ministry data." Should we add a third source toggle (Aura / USDA / Local Ministry), or is local ministry data a future phase?

5. Reports integration: Marwan said "Aura ensures all reports are available within 24 hours of publication." Should the S&D board link directly to the underlying source reports (USDA, local ministry), or keep them separate in the Reports section?

6. Divergence threshold: At what percentage difference should the red divergence alert trigger? Is this fixed or does it vary by commodity?

Layout, sidebar, right panel, basic WASDE table structure.

Month navigator for comparing USDA forecasts across months (e.g. May vs Aug). Δ Change column + 6M trend sparkline show how projections evolve. Aura Δ and Signal columns are shown here as an option — may belong in S&D Overlay instead.

Clarifying Questions

1. WASDE scope: You described WASDE as "comparing forecasts across different months (e.g. August vs May) to see how projections evolve." Should WASDE stay purely USDA month-over-month, or do you also want Aura forecasts shown here? (The S&D Overlay already handles Aura vs USDA comparison.)

2. Month navigator range: USDA starts publishing in May. Should the navigator show the full May → current month progression for the selected crop year, or allow comparing any two arbitrary months?

3. Aura's January head-start: Aura forecasts from January, 4 months before USDA's May start. Should the WASDE tool surface this advantage — e.g. showing Aura's Jan–Apr estimates alongside USDA's May+ data?

4. Livestock in WASDE: You mentioned livestock (chicken, cattle, pork) needs improvement. Does WASDE currently include livestock data, or is that only in the S&D board? If both, should the redesign treat them differently?

5. Multi-month view: The wireframe shows a two-month comparison (Feb vs Mar). Would clients prefer a full timeline view (May → Jun → Jul → ... → current) to see the trajectory at a glance?

6. Export: Do clients export WASDE comparison data to Excel/PDF for their own reporting?

What stays the same

Map, markers, sidebar, layer toggles.

What's new

Time range slider, region detail card with sparklines and commodity impact on country click.

Clarifying Questions

1. Four-tab structure: You currently have Comparison (observed temp/precip, daily), Weather (forecast from ECM + GFS), Satellite (NDVI, soil moisture, weekly), and Rivers (river levels with colour system). Should we keep all 4 tabs but redesign each for clarity, or consolidate into fewer views?

2. Rivers tab: The rivers tab highlights transport disruption risk via colour-coded river levels. How heavily used is this vs the other 3 tabs? Should it stay prominent or become a secondary layer toggle?

3. ECM vs GFS: The Weather tab shows forecasts from both ECM and GFS. Do clients compare these side-by-side, or prefer one as default with the other as optional overlay?

4. Density problem: You said the section is "too dense and difficult to navigate." For the 5–10% of clients who can read raw weather data, what's the minimum they need on first view? For the other 90%, should the default show only Aura's risk interpretation (low/medium/high) with an option to drill into raw data?

5. Time range: Are 7D / 14D / 30D / 90D the right intervals for the time slider, or does the met team use different forecast periods?

6. Satellite update cadence: Satellite data (NDVI, soil moisture) updates weekly. Is this sufficient for client needs, or is there demand for higher frequency?

What changes

Sidebar icons (weather/tractor/chart) removed. Variety radio buttons removed from sidebar. Both replaced by labelled tabs + dropdown in the slide-up panel. Same pattern for all commodities.

How variety works

If commodity has sub-types (Coffee → Robusta/Arabica, Sugar → Sugarbeets/Sugarcane), a dropdown appears in the panel header. Default: "All Varieties." If no sub-types exist (Corn, Wheat, etc.), dropdown is hidden. One pattern everywhere.

How view tabs work

☀ Weather | 📊 S&D | 💹 Market — always visible as labelled tabs. For livestock (Cattle, Hog), Weather tab either hidden or shows "N/A — not weather-dependent." Tabs replace the mystery sidebar icons.

Effort

~3-4 days. Refactor slide-up panel to support tabs. Add variety dropdown data. Remove sidebar icon logic. Net reduction in code complexity — three patterns become one.

Clarifying Questions

1. Variety data structure: Are Robusta/Arabica and Sugarbeets/Sugarcane stored as separate commodities or sub-types of a parent? This affects the dropdown logic.

2. Wheat varieties: Spring/Winter/Durum currently use map colour coding. Should these move to the dropdown too, or stay as a map legend since they're geographically distinct?

3. Livestock weather: Should Cattle/Hog have a weather tab at all? If yes, what data would it show (feed crop conditions, pasture conditions)?

4. Default view per commodity: Should all commodities default to Weather tab, or should it vary (e.g., Cattle defaults to S&D)?
