Nav, sidebar, map (visible underneath), right panel all unchanged. Banner doesn't block the workspace — it pushes content down.

What's new

Expandable signal banner under nav. Collapsed: commodity name, global risk, production, price, Aura divergence. Expanded: region cards (USA/Brazil/Argentina/Ukraine) with temp, precip, production + bottom metrics bar. Content changes when you click a different commodity in the sidebar.

Effort

Questions for Marwan

1. Global signal aggregation: How is the global risk level calculated? Production-weighted average of country signals, worst-case-takes-precedence, or manually set by the met team?

2. CMS / backend: Do meteorologists use an existing admin tool to set country signals? Can we get API access to it, or is the data in spreadsheets/emails?

3. Update frequency: How often do signals change — daily, weekly? Does the met team batch updates or update per-country as conditions shift?

4. CME price feed: Does Aura already pull commodity futures prices, or would this be a new integration?

5. Divergence benchmark: Should Aura Divergence compare against USDA's latest WASDE, or another source (e.g. IGC, local gov estimates)?

6. Threshold rules: Are temp/precip colour thresholds (green/orange/red) standardised, or do they vary by crop and region?

What changes

Sidebar icons (weather/tractor/chart) removed. Variety radio buttons removed from sidebar. Both replaced by labelled tabs + dropdown in the slide-up panel. Same pattern for all commodities.

How variety works

If commodity has sub-types (Coffee → Robusta/Arabica, Sugar → Sugarbeets/Sugarcane), a dropdown appears in the panel header. Default: "All Varieties." If no sub-types exist (Corn, Wheat, etc.), dropdown is hidden. One pattern everywhere.

How view tabs work

☀ Weather | 📊 S&D | 💹 Market — always visible as labelled tabs. For livestock (Cattle, Hog), Weather tab either hidden or shows "N/A — not weather-dependent." Tabs replace the mystery sidebar icons.

Effort

~3-4 days. Refactor slide-up panel to support tabs. Add variety dropdown data. Remove sidebar icon logic. Net reduction in code complexity — three patterns become one.

Questions for Marwan

1. Variety data structure: Are Robusta/Arabica and Sugarbeets/Sugarcane stored as separate commodities or sub-types of a parent? This affects the dropdown logic.

2. Wheat varieties: Spring/Winter/Durum currently use map colour coding. Should these move to the dropdown too, or stay as a map legend since they're geographically distinct?

3. Livestock weather: Should Cattle/Hog have a weather tab at all? If yes, what data would it show (feed crop conditions, pasture conditions)?

4. Default view per commodity: Should all commodities default to Weather tab, or should it vary (e.g., Cattle defaults to S&D)?
